# Historias de Usuario

## Plataforma

- `US-001`: Como administrador interno, quiero gestionar tenants y companies para que cada cliente opere en un scope controlado.
- `US-002`: Como usuario de plataforma, quiero un único login para acceder a los productos habilitados para mi rol.

## Workforce

- `US-003`: Como administrador cliente, quiero crear y actualizar trabajadores para que la plataforma tenga un único registro maestro por persona.
- `US-004`: Como administrador cliente, quiero registrar vehículos para que acceso y acreditación usen el mismo dato maestro.

## Acreditación

- `US-005`: Como operador contratista, quiero subir documentos requeridos para que trabajadores y vehículos puedan acreditarse.
- `US-006`: Como auditor, quiero aprobar o rechazar documentos enviados para que el estado de acreditación sea confiable.

## Control de Acceso

- `US-007`: Como guardia de acceso, quiero que el sistema decida acceso en tiempo real para mantener la operación de ingreso controlada.
- `US-008`: Como supervisor, quiero eventos y sesiones trazables para investigar incidentes.

## Cumplimiento de Contratistas

- `US-009`: Como analista interno de cumplimiento, quiero gestionar observaciones y alertas para seguir incumplimientos de contratistas.
- `US-010`: Como stakeholder mandante, quiero reportes ejecutivos para evaluar estado de cumplimiento.

## Certificación

- `US-011`: Como analista de certificación, quiero mover un caso por etapas definidas para que la certificación sea controlada y auditable.

## RRHH

- `US-012`: Como analista de RRHH, quiero seguir beneficios y capacitaciones para mantener la información laboral completa.

## Karin

- `US-013`: Como investigador Karin, quiero acceso restringido a casos para proteger la confidencialidad.
